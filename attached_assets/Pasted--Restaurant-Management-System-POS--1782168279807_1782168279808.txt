أنا بحاجة إلى بناء نظام متكامل لإدارة المطاعم (Restaurant Management System) مع نقاط بيع (POS) وقاعدة بيانات محلية (Local Database).

يرجى تصميم وبرمجة هذا النظام وفق المواصفات التالية:

1. المتطلبات العامة (General Requirements):
نوع التطبيق: تطبيق سطح مكتب (Desktop App) أو تطبيق ويب يعمل محلياً (Offline-first).

قاعدة البيانات: قاعدة بيانات محلية (SQLite / MySQL / PostgreSQL) مع إمكانية عمل نسخ احتياطي.

واجهة المستخدم: باللغة العربية (مع دعم الاتجاه من اليمين إلى اليسار RTL).

نظام صلاحيات: أدمن، مدير، كاشير (موظف استقبال).

طباعة: دعم الطباعة المباشرة على طابعة حرارية (Thermal Printer) لإيصالات الطلبات والفواتير.

2. لوحة تحكم رئيسية (Dashboard):
عرض ملخص سريع: إجمالي المبيعات اليومية، عدد الطلبات، عدد العملاء، العناصر الأكثر مبيعاً.

رسم بياني يوضح حركة المبيعات خلال الأسبوع أو الشهر.

إشعارات عند اقتراب المخزون من النفاد.

3. إدارة المنتجات والأصناف (Menu & Products):
إضافة، تعديل، حذف الأصناف.

تصنيف الأصناف (مقبلات، مشروبات، أطباق رئيسية، حلويات).

إمكانية إضافة خيارات للصنف (مثل: حجم، إضافات، درجة السكر) مع تسعير مختلف لكل خيار.

رفع صورة لكل صنف.

تحديد سعر الشراء وسعر البيع.

تفعيل خاصية "متوفر/غير متوفر" (Out of Stock).

4. نظام نقاط البيع (POS - Cashier):
شاشة رئيسية للموظف (الكاشير) تحتوي على:

أزرار الأصناف مقسمة حسب التصنيفات.

عربة الطلب (Order Cart) تعرض المنتجات المضافة والكميات والسعر الإجمالي.

إدخال رقم الطاولة أو رقم الطلب الخارجي (Delivery).

إضافة خصم على الفاتورة (نسبة مئوية أو قيمة ثابتة).

إضافة ضريبة المبيعات (VAT) تلقائياً.

إتمام الدفع بعدة طرق (نقدي، بطاقة ائتمان، تحويل بنكي) مع حساب الباقي (Change).

طباعة إيصال الطلب مباشرة بعد الدفع.

إمكانية إلغاء طلب أو استرجاع منتج (Refund).

5. إدارة الطلبات (Orders Management):
عرض جميع الطلبات السابقة مع الفلترة حسب (التاريخ، الموظف، طريقة الدفع).

عرض تفاصيل كل طلب.

إمكانية إعادة طباعة الإيصال لأي طلب سابق.

6. إدارة المخزون والمستودع (Inventory):
ربط الأصناف بالمخزون (تحديث تلقائي عند البيع).

جرد المخزون يدوياً (إدخال الكمية الفعلية ومقارنتها بالمتوقعة).

تقرير عن الأصناف منخفضة المخزون.

إدخال مشتريات جديدة (Purchase Orders) لإضافة كميات للمخزون.

7. إدارة الموظفين والعملاء:
إضافة موظفين مع صلاحيات محددة.

تسجيل العملاء (اسم، هاتف، عنوان) مع عرض تاريخ آخر طلب.

نظام نقاط ولاء (Loyalty Points) للعملاء (اختياري).

8. التقارير والإحصائيات (Reports):
تقرير المبيعات اليومية / الأسبوعية / الشهرية.

تقرير الأصناف الأكثر ربحية.

تقرير عن المدفوعات النقدية مقابل البطاقات.

تقرير حركة المخزون (رصيد أول المدة، وارد، منصرف، رصيد آخر المدة).

تصدير التقارير بصيغة PDF أو Excel.

9. الإعدادات والخيارات (Settings):
إعدادات عامة: اسم المطعم، العنوان، رقم الهاتف، الشعار (يظهر في الإيصال).

إعدادات الضريبة (نسبة الضريبة، تفعيل/إلغاء).

إعدادات الطابعة (نوع الطابعة، حجم الورق).

إدارة المستخدمين وتغيير كلمات المرور.

عمل نسخة احتياطية من قاعدة البيانات (Backup) واستعادتها (Restore).

10. متطلبات تقنية إضافية:
استخدام SQLite كقاعدة بيانات محلية (لتكون خفيفة ولا تحتاج تثبيت).

أو استخدام MySQL مع إمكانية التشغيل عبر XAMPP.

البرمجة باستخدام Python (Tkinter / PyQt) أو C# (WPF / WinForms) أو PHP (Laravel) مع واجهة مستخدم غنية.

يجب أن يكون الكود نظيفاً، مؤمناً، ومقاوماً للأخطاء (Error Handling).

توفير واجهة API مستقبلية لربطها بتطبيق جوال (اختياري).

المطلوب النهائي:

تقديم الكود المصدري الكامل مع هيكل قاعدة البيانات (SQL Schema).

تقديم ملف تعليمات (User Manual) بسيط يشرح كيفية التنصيب والتشغيل.

أن يكون النظام جاهزاً للتشغيل الفوري في مطعم حقيقي.
